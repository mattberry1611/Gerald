FEATURES ADDED
--------------
1. BOTTOM OVERFLOW FIX
   - Root cause: hard-coded 20px bottom padding in bottom bar → RenderFlex overflow
     when Android keyboard is visible (keyboard shrinks body, 20px wasted)
   - Fix: padding is now dynamic via MediaQuery.viewInsets.bottom
   - ActivityLog hides when keyboard is up to give more room to conversation list

2. COMMAND QUEUE SYSTEM
   - New commands sent while Gerald is busy are queued (not dropped)
   - Queue drains automatically: next task starts when current completes or is rejected
   - Status panel shows amber "N tasks queued" indicator
   - Activity log entries: "Queued [N]: ...", "Processing queued task (N remaining)"
   - Queue is cleared on conversation clear

3. IMAGE UPLOAD SUPPORT (V1)
   - Paperclip button in bottom bar opens Camera / Gallery picker
   - Selected image displayed in conversation thread as a preview
   - Message stores local imagePath for future backend upload
   - Image quality capped at 85% / 1280px wide to reduce file size
   - Error fallback shown if image file is missing/unreadable
   - Backend upload NOT yet implemented (V1 scope: select + preview only)
