MODE A BEHAVIOUR
----------------
- Chip label: "A  PUSH-TO-TALK" (bottom of main screen)
- 100x100 green mic circle button
- Tap to start recording; speech captured until 3-second pause or 60-second max
- Release / second tap sends the recognized text to the backend
- No auto-resume — one command at a time
- Pulsing animation while recording
- Haptic feedback on tap

MODE B BEHAVIOUR
----------------
- Chip label: "B  CONVERSATION" (bottom of main screen)
- Activate by tapping the B chip; deactivate by tapping A chip or the END button
- On activate: STT starts listening automatically
- User speaks; on pause detection (3 seconds of silence), command is sent
- After backend responds: TTS speaks (if enabled), then STT restarts automatically
- 80x80 ring indicator shows LISTENING (red pulse) / PROCESSING (blue pulse) / READY (green)
- STT timeout (60s no speech): auto-restarts listening instead of stopping
- Persisted to SharedPreferences — remains on between app restarts

TEXT-TO-SPEECH BEHAVIOUR
-------------------------
- Package: flutter_tts ^4.0.0 (uses Android on-device TTS engine)
- Toggle: Settings → Voice → "Speak Gerald responses"
- Persisted to SharedPreferences
- Fires for every gerald message received via _readResult()
- Speaks the full response text with a 120-second timeout guard
- In Mode B: TTS completes BEFORE listening resumes (speech overlapping is prevented)
- TTS stopped when disabling the toggle or exiting conversation mode

REMAINING WORK FOR LOCKED-SCREEN CONVERSATION MODE (V2)
---------------------------------------------------------
- Android Foreground Service with persistent notification (keeps process alive)
- FOREGROUND_SERVICE + FOREGROUND_SERVICE_MICROPHONE permissions (Android 14+)
- Native Kotlin SpeechRecognizer binding (not Flutter plugin) to keep mic active
- WAKE_LOCK to prevent CPU sleep during background sessions
- Optional offline wake-word detection (e.g. Vosk) to avoid always-on recording
- MethodChannel to bridge native recognition events back to Flutter
- UI for enabling background mode with appropriate permission explanations
