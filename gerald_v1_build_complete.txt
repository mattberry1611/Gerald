REMAINING WORK BEFORE RELEASE
─────────────────────────────────────────────────────────────────

1. NPM INSTALL
   cd gerald-app && npm install
   Installs all declared dependencies (Expo, RN, Zustand, Axios,
   @react-native-voice/voice, expo-notifications, react-native-paper, etc.)

2. FIREBASE SETUP
   - Create Firebase project at console.firebase.google.com
   - Enable Cloud Messaging (FCM)
   - Download google-services.json → place at gerald-app/google-services.json
   - Add "googleServicesFile": "./google-services.json" to app.json under android
   - Replace "REPLACE_WITH_EAS_PROJECT_ID" in app.json with real EAS project ID
   - Add firebase-admin to gerald_bridge.py to actually send push notifications
     (currently /register-device stores tokens but sending is not implemented)

3. APP ASSETS
   - Add gerald-app/assets/icon.png (1024×1024)
   - Add gerald-app/assets/adaptive-icon.png (1024×1024, Android adaptive)
   - Add gerald-app/assets/notification-icon.png (96×96, white on transparent)
   Without these, EAS Build will use Expo defaults but app.json references them.

4. EAS BUILD SETUP
   - eas login (Matt's Expo account)
   - eas build:configure (links project to EAS)
   - eas build --platform android --profile preview
   This produces a signed APK for sideloading.

5. NETWORK CONFIGURATION
   - Update Settings screen base URL to Matt's PC's LAN IP (e.g. 192.168.1.x:8000)
   - Ensure gerald_bridge.py is running (uvicorn gerald_bridge:app --host 0.0.0.0 --port 8000)
   - Test GET /status from phone browser before first PTT use

6. WIRE send-to-claude-code TO claude_desktop_controller.py
   Currently POST /send-to-claude-code writes approval to outbox but does NOT
   inject into the live Claude Code terminal window. Wire it to
   claude_desktop_controller.py's send_to_claude_code() function for full loop.

7. PTT VOICE TESTING
   @react-native-voice/voice requires a native build (not Expo Go).
   Run `eas build --profile development` for a dev client build,
   or `eas build --profile preview` for a testable APK.

8. NOTIFICATION PERMISSION FLOW
   Android 13+ requires POST_NOTIFICATIONS runtime permission. This is
   requested via expo-notifications but should be tested on a physical device.
